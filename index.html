Slide 2 — Quick Recap (3 minutes)

Slide Content

Last class: - AWS Console - Regions - Availability Zones

Question: What are we launching when we launch a cloud server?

Answer: Virtual Machine

Speaker Script

Before starting today’s topic let’s quickly review what we covered last class.

We explored the AWS console and discussed how AWS divides its infrastructure into regions and availability zones.

Regions represent geographic areas, and availability zones represent separate data centers designed for reliability.

Now here’s an interesting question.

When you click “Launch Instance” in AWS, what are you actually getting?

Is AWS giving you an entire physical computer?

In most cases the answer is no.

Instead AWS creates something called a Virtual Machine.

That is what we will explore today.

Slide 3 — The Problem Before Virtual Machines (3 minutes)

Slide Content

Earlier companies had: - One physical server - One application - One OS

Problems: - Hardware waste - Expensive servers - Hard to scale - Slow deployment

Speaker Script

Before virtualization existed, companies used physical servers directly.

If a company wanted to run an application they would purchase a server, install an operating system, and run the application on that machine.

Usually each server ran only one application.

This approach caused many problems.

Hardware was often underutilized. Servers were expensive. Scaling required purchasing more hardware. And deploying a new server could take days.

Clearly this system was inefficient.

Slide 4 — Example of Hardware Waste (3 minutes)

Slide Content

Example server: 32 CPU cores 128 GB RAM

Application usage: 4 CPU 8 GB RAM

Speaker Script

Imagine buying a powerful server with 32 CPU cores and 128 GB of RAM.

But your application only uses 4 CPU cores and 8 GB of RAM.

That means most of the hardware is doing nothing.

Yet the company still paid for the full server.

Now imagine thousands of companies doing this.

That means huge amounts of computing power are wasted.

Virtualization was created to solve this exact problem.

Slide 5 — What is a Virtual Machine (3 minutes)

Slide Content

Virtual Machine = software version of a computer

Each VM has: - OS - CPU allocation - RAM allocation - Disk - Network

Speaker Script

A virtual machine is essentially a computer created using software.

It runs inside a physical server but behaves like a completely independent system.

Each VM has its own operating system, CPU resources, memory, storage, and networking.

From the user’s perspective it behaves exactly like a real computer.

Slide 6 — Simple Analogy (3 minutes)

Slide Content

Physical server = Apartment building

Virtual machines = Apartments

Speaker Script

A helpful analogy is to imagine a physical server as an apartment building.

The building itself is the hardware.

Each apartment is a virtual machine.

Each apartment is separate and independent, but they all share the same building infrastructure.

Slide 7 — Hypervisor (3 minutes)

Slide Content

Hypervisor responsibilities: - Divide hardware - Create VMs - Isolate machines

Examples: VMware Hyper-V KVM

Speaker Script

The technology that makes virtualization possible is called a hypervisor.

A hypervisor manages the hardware resources and distributes them among multiple virtual machines.

It also isolates the machines so that one VM cannot interfere with another.

Slide 8 — Architecture Overview (3 minutes)

Slide Content

Physical Hardware → Hypervisor → Virtual Machines

Speaker Script

At the bottom we have the physical server hardware.

On top of the hardware runs the hypervisor.

On top of the hypervisor we run multiple virtual machines.

Each virtual machine behaves like an independent server.

Slide 9 — What Runs Inside a VM (3 minutes)

Slide Content

Inside a VM: Linux Windows Applications Databases Web servers

Speaker Script

Inside a virtual machine you can install anything you would normally install on a computer.

For example Linux servers, web servers, databases, or enterprise software.

From inside the VM everything feels like a real machine.

Slide 10 — Why Cloud Providers Use VMs (3 minutes)

Slide Content

Benefits: Efficient hardware usage Fast provisioning Better isolation Flexible resources

Speaker Script

Cloud providers rely heavily on virtualization because it allows them to maximize hardware usage.

Instead of giving customers entire physical servers they create multiple virtual machines.

This also allows them to create servers within seconds.

Slide 11 — Example AWS EC2 (2 minutes)

Slide Content

Launching EC2 = launching a virtual machine

Choose: CPU RAM OS Storage

Speaker Script

When you launch an EC2 instance on AWS you are actually creating a virtual machine.

You choose the CPU size, memory, operating system, and storage.

Then you connect to the machine using SSH.

Slide 12 — Advantages of Virtual Machines (2 minutes)

Slide Content

Isolation Efficiency Scalability Flexibility

Speaker Script

Virtual machines provide strong isolation, efficient hardware usage, easy scaling, and flexibility to run different operating systems.

Slide 13 — Transition to Storage (2 minutes)

Slide Content

Computers need storage for: OS Applications Databases User files

Speaker Script

Now that we understand computing resources we need to talk about something equally important: data storage.

Every application stores data somewhere.

Slide 14 — What is Storage (2 minutes)

Slide Content

Storage = where data is permanently saved

Examples: Files Images Backups Logs

Speaker Script

Storage is simply the place where data is stored permanently.

Different applications require different storage solutions.

Slide 15 — Three Major Storage Types (2 minutes)

Slide Content

Block Storage Object Storage File Storage

Speaker Script

Most cloud platforms provide three major storage models.

Each is designed for different types of workloads.

Slide 16 — Block Storage (3 minutes)

Slide Content

Used for: VM disks Databases Operating systems

Example: AWS EBS

Speaker Script

Block storage works similar to a hard drive.

Data is divided into blocks and accessed quickly.

This makes it ideal for operating systems and databases.

Slide 17 — Object Storage (3 minutes)

Slide Content

Used for: Images Videos Backups

Example: AWS S3

Speaker Script

Object storage stores data as objects instead of blocks.

It is extremely scalable and perfect for storing large amounts of unstructured data.

Slide 18 — File Storage (3 minutes)

Slide Content

Folders and files

Example: AWS EFS

Speaker Script

File storage organizes data using traditional folder structures.

This is useful when multiple servers need to share the same files.

Slide 19 — Simple Comparison (2 minutes)

Slide Content

Block → VM disks Object → media and backups File → shared folders

Speaker Script

Each storage type solves a different problem.

Choosing the right storage type is an important architectural decision.

Slide 20 — Real World Example (3 minutes)

Slide Content

Web application architecture

VM → server Block → OS + DB Object → images File → shared logs

Speaker Script

A real application might use all three storage types together along with virtual machines.
